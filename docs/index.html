How Aura Works

A coding loop built for inspection.

Aura separates planning from execution so the agent has a job, the user has control, and every meaningful step can be reviewed.

01

Planner

Reads your workspace, repo map, search results, and relevant files. Produces a human-readable spec before code is touched.

02

Worker

Receives the approved spec and executes against the real repository with controlled file tools and validation hooks.

03

Reviewable diffs

Every proposed write is shown as a diff. Approve, reject, approve all, or reject all before anything lands on disk.

04

Validation

Run checks, inspect failures, retry when appropriate, and stop cleanly if recovery is not the right move.

05

Receipts

See tool calls, files changed, validation results, and token cost so the session has an audit trail.

Product Proof

The desktop app is the product.

Aura is not a chat wrapper. The cockpit is built around dispatch, work state, diff approval, and completion receipts.

Aura Companion

A phone-side command center for your desktop agent.

Companion lets you steer Aura from your phone while the desktop stays responsible for the real workspace. Browse projects, send Planner messages, dispatch specs, watch live execution, and review receipts away from the keyboard.

  • ✓Remote control surface, not a separate mobile IDE.
  • ✓Project and thread browsing from the phone.
  • ✓Desktop Worker still performs the actual repo work.

Model Access

Start easy, or bring your own keys.

Aura supports a hosted provider path through Aura Credits and a user-owned provider path through BYOK. Use either one, or mix providers between Planner and Worker.

Aura Credits

Hosted path for a faster first run.

Buy credits, select Aura as the Planner or Worker provider, and start without managing API keys. Balance and session spend stay visible in the app.

BYOK

User-owned provider control.

Connect your own keys for supported providers such as DeepSeek, OpenAI, Anthropic, Gemini, and OpenRouter. Your model path stays configurable per role.

Drones

Bounded background automation with the same accountability.

Drones are reusable automation cards for repeatable repo work. They can run read-only checks, command tasks, or bounded harness laps through Aura's Planner/Worker loop.

Write policies Read-only, ask-before-writes, or normal diff approval.
Bounded laps Guardrails such as clean-worktree checks, protected paths, and changed-file caps.
Saved receipts Every run leaves status, elapsed time, errors, and output metadata.
