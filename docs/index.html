AI coding agents need receipts, not vibes.

How Aura works

Planner writes a bounded work artifact. Worker executes through tools. You review diffs. Validation runs. Aura leaves a receipt.

  1. Plan. Aura reads the workspace and writes a structured spec before code is touched.
  2. Dispatch. The Worker gets a bounded job instead of a vague chat wish.
  3. Review. File edits are visible as diffs before they land.
  4. Validate. Aura runs checks and reports what happened.
  5. Receipt. You get the trail: changed files, tool calls, validation, and cost.
