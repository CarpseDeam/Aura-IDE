## Problem (from user screenshots)

1. Chat text is too low contrast — body copy looks gray on dark gray, hard to read for long sessions.
2. User messages and assistant messages look near-identical: same dark card, same inner background, only the small "You" / "Aura" label distinguishes them. After a few turns the eye can't separate the conversation flow.
3. Tool-call cards stack visually well but the assistant message body and tool cards blur together — needs visual hierarchy: assistant *message* vs assistant *tool calls under the message*.

## Fixes

### 1. Body text contrast

In `theme.py`, raise the primary chat text color. Current value is probably around `#9ca3af` or similar mid-gray. Change to something near `#e5e7eb` (light cool gray) for assistant body content. User message body can be slightly different (see below) but should also hit at least `#d4d4d8`. Keep secondary/muted text (the "Thinking" label, tool-call meta, timestamps if any) at the current muted level so it stays subordinate.

Verify line-height is comfortable — if it's currently 1.0 / default, bump to 1.4-1.5 for body paragraphs. Code blocks already get monospace treatment from Pygments; leave those.

Font size: if body is currently 12-13px, leave it; if smaller, bump to 13. Don't go bigger than 14 — this is a dense tool, not a reading app.

### 2. Distinguish user vs assistant messages

Goal: the user should be able to scan the transcript and see the conversation rhythm at a glance, even from across the room.

Two visual axes to use (use both):

**Color tint:**
- User card: a slightly cooler / bluer dark background (e.g. `#1e2536` or similar — subtle blue tint, NOT a saturated blue), with a thin left border in the accent color (use whatever accent the theme already defines — looks orange/amber from the screenshots).
- Assistant card: warmer neutral dark (e.g. `#1f1f23` — nearly the existing color but slightly different), no left border, OR a left border in a muted secondary color.

The two backgrounds should be clearly different at a glance but neither should be loud. Test by squinting at the screen — you should still see two distinct colors.

**Alignment / shape:**
- Keep both cards full-width (don't go to chat-bubble alignment — this is a tool, not iMessage). 
- User cards: maybe slightly inset on the right (margin-right: ~40px) so they feel like input.
- Assistant cards: full width or slightly inset on the left.

This combination — different tint + different inset — gives a strong rhythm without being cartoonish.

**Role labels:**
- "You" label: in the accent color, bold, 11-12px, more presence than current.
- "Aura" label: in a secondary/muted color, same size, slightly less prominent.

### 3. Hierarchy: assistant message vs its tool calls

Right now the assistant body and the tool-call cards underneath are both inside the same visual container, distinguished only by the green tool-call card styling. From the screenshot, after the model gives a long answer, a wall of tool cards stacks below and the eye loses track of which call belongs to which turn.

Fix:
- Indent tool-call cards ~16-20px from the left edge of the assistant card content area, so they read as "children" of the assistant turn.
- Add a thin vertical line on the left of the indent (1px, muted color) connecting the tool calls visually as a group — like a thread/branch indicator.
- Reduce vertical spacing between consecutive tool cards (probably currently 8-12px gap, drop to 4-6px) so the cluster reads as one unit rather than disconnected items.
- Keep the green accent on tool cards but reduce its saturation slightly so they feel like supporting info, not the main event. The body text of the assistant should still feel like the primary content of the turn.

### 4. Reasoning section polish

The "▶ Thinking" collapsed header is currently good but the click affordance could be clearer. Make it:
- Slightly muted background tint when hovered
- Cursor changes to pointer on hover
- The caret rotates from ▶ (collapsed) to ▼ (expanded) — if it doesn't already

### 5. Spacing pass

- Vertical gap between consecutive turns (user → assistant or assistant → user): currently looks ~16-20px from screenshots. Bump to ~28-32px so turns visually breathe.
- Inner padding of message cards: looks tight around the role label specifically — give the role label ~8px below it before the body text starts.
- Don't change anything in the input panel or toolbar — those look fine.
